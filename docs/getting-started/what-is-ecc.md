# ECC 是什么

`Everything Claude Code`（简称 **ECC**）是来自于 Anthropic 黑客马拉松获胜者的 **Claude Code 完整配置与开发辅助系统**。

它不仅仅是一系列零散的配置文件，而是一整套为 AI 智能体（Agentic Coding）量身定制的**开发提效操作系统**。通过提供开箱即用的智能体、最佳实践、规则约束和自动化脚本，帮助开发者在与 Claude Code（以及 Cursor、Gemini、Codex 等 AI 智能体）协同开发时，获得最佳的 Token 消耗比、极高的任务成功率以及严密的安全防护。

---

## 核心能力：ECC 能为你做什么？

在日常开发中，使用原生 Claude Code 经常会遇到：**AI 容易遗忘上下文、反复犯同一种错误、无法做复杂的长任务拆解、或者随意改动不相关的代码**。

ECC 围绕这几个痛点，提供了一整套解决方案：

1. **子任务委派 (Agents)**：内置 36 个专用子智能体，如规划者 (`planner`)、架构师 (`architect`)、代码审查员 (`code-reviewer`) 等。主智能体可根据当前任务将工作委派给专业子代理，大幅提升长任务成功率。
2. **场景工作流 (Skills)**：包含 271 个特定领域或特定框架的最佳实践（如 TDD 工作流、Django 模式、React 模式等），让 AI 按照业内最高标准编写代码，拒绝过度设计与冗余代码。
3. **本能与持续学习 (Continuous Learning)**：引入了记忆持久化钩子与直觉学习系统。AI 在开发中犯过的错、总结出的模式会被自动固化，在下次会话中自动加载，真正做到“AI 越用越聪明”。
4. **自动化防御 Gate (Hooks & Rules)**：通过设定各种工具事件的拦截钩子（如禁止提交包含 `console.log` 的代码、自动跑类型检查）和规范规则（Rules），把控最终代码输出质量。
5. **安全审计护航 (AgentShield)**：内置 AgentShield 静态与动态安全审查工具，能够自动扫描配置漏洞、避免 AI 产生潜在的数据泄漏、指令注入等安全漏洞。

## ECC 的五大基石 (Architecture)

ECC 所有的组件都在围绕以下五个层级协作运转：

*   **Agent (智能体)**：限定特定职责与工具集的专属 AI 角色，帮助解决“单 Agent 容易跑偏”的问题。
*   **Skill (技能)**：可重用的工作流规范与经验文件，规范 AI 的操作逻辑。
*   **Command (自定义命令)**：直接扩展 Claude Code 的命令空间，提供 `/plan`、`/code-review`、`/checkpoint` 等快捷操作。
*   **Hook (钩子)**：由特定的工具调用（如 `Edit` / `run_command`）触发的自动化脚本，实现前置审计与自动修复。
*   **Rule (规则)**：放置在 `~/.claude/rules/` 目录下的多语言开发指南，AI 会在执行过程中始终作为首要约束遵守。

- 清晰理解**上游副本的文件结构**与维护方式。
